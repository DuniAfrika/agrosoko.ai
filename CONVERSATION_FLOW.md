## Priority Rules

When input could match multiple patterns, apply in this order:

```
1. API Error Check (always first)
2. Support/Help Keywords
3. "join" command
4. "start" command (valid county)
5. "start" command (invalid county)
6. "YES" response
7. Invalid response (after prices)
8. Unknown command (catch-all)
```
